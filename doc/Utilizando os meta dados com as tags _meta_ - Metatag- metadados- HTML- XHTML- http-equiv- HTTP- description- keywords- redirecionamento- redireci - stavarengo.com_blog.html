Utilizando os meta dados com as tags <meta>

Postado por Stavarengo em 29/mai/2011 Metatag, metadados, HTML, XHTML, http-equiv, HTTP, description, keywords, redirecionamento, redireci

Os meta dados são estruturas de dados incorporadas ao código XHTML. Estas estruturas armazenam diferentes tipos de informações, como descrição do conteúdo da página, autor, data de criação, linguagem, entre muitas outras coisas. 

Os meta dados são criados com a tag <meta> que devem ser adicionadas ao cabeçalho da página, entre as tags <head> e </head> . A tag <meta> pode ter dois atributos: 1) http-equiv ou name ; 2) content . 

Exemplo 1 - Inserindo meta dados com a tag <meta>

?
1
2
<meta http-equiv="Content-type" content="text/html; charset=UTF-8" />
<meta name="author" content="Rafael Stavarengo" /> 

Tipo de meta tags

http-equiv

Tags <meta> declaradas com o atributo http-equiv  equivalem aos cabeçalho da respostas HTTP. No exemplo 1 você pode ver como definir o Content-type da página usando a tag <meta> e o atributo http-equiv . Com isto, se o Content-type não estiver presente no cabeçalho da resposta, o browser deve usar a informação definida no atributo content desta tag.

Nota: Tags com http-equiv são ignoradas por servidores proxy e por alguns browsers antigos, por este motivo é recomendado usar os cabeçalhos HTTP equivalente.

Você pode ler sobre os cabeçalhos HTTP nestes links: HTTP 1.0 e HTTP 1.1.

name

Tags <meta> declaradas com o atributo name NÃO são equivalentes aos cabeçalhos HTTP, apesar de que esta diferença em relação as tags com atributo http-equiv  ser ignorada por muitos desenvolvedores.

Dicas de uso

Antes de ver a lista de meta tags existentes, você precisa saber que o uso da tag <meta> tem seus pros e contras, apesar de que o uso correto e consciente pode praticamente anular seu lado negativo. Veja algumas dicas.

Escrevas descrições simples e keywords concisas

Descrições muito longas na meta tag description  e palavras-chave em excesso na meta tag keywords  serão ignoradas por alguns buscadores, inclusive, alguns deles consideram apenas as seis primeiras keywords. Além disto, alguns sistemas de indexação podem também identificar o chamado "metatag spamming", onde as palavras-chave são repetidas várias vezes. Neste caso eles podem penalizar o site diminuindo sua posição no ranking de resultados.

Palavras-chave mais importantes devem vir antes

Uma vez que alguns sistemas de indexação leem somente algumas das keywords é importante listar as mais importantes primeiro.

Evite redirecionar usando a meta tag REFRESH

Esta era uma técnica muito comum antigamente, mas, hoje em dia deve ser evitada porque ela tende a confundir os sistemas de indexação, além de também confundir o usuário, especialmente aqueles que estejam usando leitores de tela.

Não abuse das meta tags

Meta tags fornecem informações usadas para categorizar, priorizar e criar um ranking de websites além de controlar seu conteúdo pelos webmasters e/ou autores. Mas você não deve abusar destes privilégios, como dito anteriormente muitos sistemas de indexação são "inteligentes" o suficiente para perceber quando há este tipo de abuso, penalizando o site em sua colocação no ranking.

Lista de metatags, atributos e especificações.

Muitos ainda não têm um conhecimento adequado sobre a maioria dos recursos oferecidos pela tag <meta> . Abaixo, segue uma lista das principais meta tags, seus valores e funcionalidades. Na lista abaixo você vai encontrar exemplos de uso, tanto para tags <meta> com atributo name, como também para aquelas com atributo http-equiv .

Nota: As palavras-chave http-equiv , name e content são case-insensitive, assim como seus valores, mas segundo as especificações do XHTML, todo o código deve estar em letras minúsculas.

Author

Define o nome do autor da página.

?
1
<meta name="author" content="Marcos" />

Copyright

Declara o direito autoral da página.

?
1
<meta name="copyright" content="© 2011 Empresa XYZ" />

Description

Contém uma descrição da página sobre o conteúdo da página.

?
1
<meta name="description" content="...descrição da página..." />

Keywords

São frases ou palavras separadas por vírgulas, que definem as palavras chaves associadas ao conteúdo da página. Normalmente, as keywords são usadas pelos motores de busca para indexar os documentos juntamente com informações encontradas no título e corpo da página.

?
1
<meta name="keywords" content="...palavras chave do documento..." />

Generator

Indica o nome do software usado para criar a página como forma de medir a popularidade do produto.

?
1
<meta name="generator" content="Dreamweaver 8" />

Revisit-After

Diz para os servidores proxy refazer o cache da página depois de um tempo específico.

Esta meta tag não faz com que os motores de busca voltem para sua página. Eles fazem isso em períodos aleatórios.

?
1
<meta name="revisit-after" content="15 days" />

Rating

Esta tag classifica a página por censura, assim como no cinema, suporta os valores:

  • General: Para qualquer idade
  • 14 years: Censura 14 anos
  • Mature: Para pessoas acima de 18 anos
?
1
<meta name="rating" content="general" />

Robots

Especifica informações de indexação para os robôs de busca, suporta os seguintes valores:

  • All: Valor default, significa vazio, o robô de busca não recebe nenhuma informação.
  • Index: Os robôs de busca podem incluir a página normalmente.
  • Follow: Robôs podem indexar a página e ainda seguir os links para outras páginas que ela contém.
  • NoIndex: Os links podem ser seguidos, mas a página não é indexada.
  • NoFollow: A página é indexada, mas os links não são seguidos.
  • None: Os robôs podem ignorar a página.
  • NoArchive (Apenas Google): A página não é arquivada.
?
1
<meta name="robots" content="all" />

Content-type

Define o tipo de conteúdo e tipo de codificação de caracteres da página (sempre deve ser usada).

?
1
<meta http-equiv="content-type" content="text/html; charset=UTF-8" />

Content-language

Declara a uma ou mais linguagens do documento. Pode ser usada pelos motores de busca para categorizar a página por idioma.

?
1
<meta http-equiv="content-language" content="pt-br, en-US, fr" />

Cache-Control

Reconhecida apenas pelo HTTP 1.1, esta tag aceita os seguintes valores:

  • Public: O browser pode armazenar o cache do conteúdo da pagina e compartilhar este cache com diferentes usuários do mesmo browser.
  • Private: É o contrário de public . O cache é armazenado para um usuário específico (OBS: é possível especificar usuários apenas no Firefox e Opera).
  • No-Cache: O browser não cria cache para a página.
  • No-Store: É feito um cache temporário, a página não é arquivada.

Nota: O uso de cache-control:no-cache  do HTTP 1.1 é equivalente ao pragma:no-cache  (veja aqui) do HTTP 1.0. Por garantia, sempre use ambas as formas caso não saiba se o servidor é ou não compatível com HTTP 1.1.

?
1
<meta http-equiv="cache-control" content="no-cache" />

Pragma no-cache

Faz com que o navegador não armazene a página em cache. A diferencia em relação a  cache-control:no-cache é que pragma:no-cache é reconhecida por todas as versões do HTTP.

?
1
<meta http-equiv="pragma" content="no-cache" />

Expires

Define a data e hora a partir do qual o documento deve ser considerado como expirado. Uma data ilegal como, por exemplo "0" é considerada como "agora". Configurar Expires  como "0" também é usado para forçar que o robô de busca faça uma nova checagem a cada visita.

Nota: Alguns robôs de busca podem apagar um documento de seu banco de dados quando o mesmo estiver expirado, outros podem marcar uma revisita.

?
1
<meta http-equiv="expires" content="Sun, 29 may 2011 14:44:09 GMT" />

Refresh

Especifica um tempo em segundos para o browser atualizar a página. Opcionalmente pode-se adicionar uma URL para a qual o browser será redirecionado. O metadado abaixo redireciona o browser para o site http://exemplo.com após 15 segundos.

?
1
<meta http-equiv="refresh" content="15;url=http://exemplo.com" />

Imagetoolbar

Elimina aquela pequena barra de opções que aparece sempre que passamos o mouse por cima de uma imagem no Internet Explorer.

?
1
<meta http-equiv="imagetoolbar" content="no" />

Content-Script-Type

Define o tipo padrão da linguagem de script do documento.

?
1
<meta http-equiv="content-script-type" content="text/javascript" />    

Content-Style-Type

Define o tipo padrão de linguagem para estilização do documento.

?
1
<meta http-equiv="content-style-type" content="text/css" />

Content-Script-Type

Define o tipo padrão da linguagem de script do documento.

?
1
<meta http-equiv="content-script-type" content="text/javascript" />    

Content-Style-Type

Define o tipo padrão de linguagem para estilização do documento.

?
1
<meta http-equiv="content-style-type" content="text/css" />

 

Projeto Dublin Core 

Na lista abaixo você encontrará algumas tag  <meta>  cujo valor do atributo  name  começa com 'DC'. Estas tags fazem parte do projeto Dublin Core iniciado em 1995, cujo objetivo, segundo os organizadores, é melhorar a indexação das páginas pelos motores de busca e dar mais flexibilidade aos autores.

Site oficial do projeto: http://www.dublincore.org 

DC.title

É o mesmo que "title", e deve conter o mesmo valor.

?
1
<meta name="DC.title" content="Título da página" />

DC.Creator

É o mesmo que "Author" e recomenda-se que tenha o mesmo valor.

?
1
<meta name="DC.creator " content="Marcos, Leonidas" />

DC.Creator.adress

E-mail para contato com o autor da página.

?
1
<meta name="DC.creator.address" content=email@exemplo.com" />

DC.Suject

É o mesmo que "Keywords" e recomenda-se que tenha o mesmo valor.

?
1
<meta name="DC.subject" content="...palavras chave do documento..." />

DC.Description

É mesmo que "Description" e recomenda-se que tenha o mesmo valor.

?
1
<meta name="DC.description" content="...descrição da página..." />

DC.Publisher

Nome da organização responsável pelo documento.

?
1
<meta name="DC.publisher" content="Stavarengo.com" />

DC.Custodian

Normalmente, o webmaster responsável pela página.

?
1
<meta name="Custodian" content="Marcos, Leonidas" />

DC.Date.Created

Data de criação da página no formato AAAA-MM-DD.

?
1
<meta name="DC.date.created" content="2011-05-29" />

DC.Date.Modified

Data da última modificação do documento, importante para buscas por data.

?
1
<meta name="DC.date.modified" content="2011-05-29" />

DC.Identifier

URL do documento.

?
1
<meta name="DC.Identifier" content="http://www.exemplo.com/article/XYZ">      

DC.Format

Especifica o tipo de dados contidos no documento. Veja a lista de valores válidos no site do projeto Dublin Core.

?
1
<meta name="DC.format" content="text/xhtml" />

DC.Type

Determina a natureza ou gênero do documento. Veja a lista de valores válidos no site do projeto Dublin Core.

?
1
<meta name="DC.type" content="text.homepage.institucional" />
